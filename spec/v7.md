## Entity

The `Entity` is the primary storage-centric primitive. It unifies schema definition, storage adapter selection, and lifecycle hooks into a single declaration.

```ts
const UserEntity = entity({
  name: "user",
  storage: "postgres",
  schema: (builder) => {
   builder.increments("id");
   builder.string("name").notNullable();
   builder.string("status").defaultTo("active");
   builder.integer("org_id").references(OrgEntity.schema.id);
   builder.timestamps(true, true);
  },
  hooks: (ctx) => {
    ctx.beforeInsert(async (data) => {
      // Logic before data hits storage
    });
    ctx.afterUpdate(async (data) => {
      // Trigger side effects (e.g., send email via Job)
      await SendEmailJob.dispatch({ to: data.email, subject: "Updated" });
    });
  },
});
```

## Logic Primitives

For logic that isn't naturally centered on a storage model (e.g., complex orchestration, background tasks, or scheduled events), Aromix provides first-class durable primitives.

### Workflows & Steps

Durable execution for complex, multi-step processes. Steps are independent and reusable.

```ts
const ProcessVideo = Workflow.step({
  name: "process-video",
  input: v.object({ path: v.string() }),
  async run(input) {
    return ffmpeg(input.path).compress();
  },
});

const VideoWorkflow = Workflow.define({
  name: "video-onboarding",
  input: v.object({ videoPath: v.string() }),
  steps: (wire) => [
    wire(ProcessVideo, {
      input: (ctx) => ({ path: ctx.input.videoPath }),
    }),
  ],
});
```

### Jobs & Crons

Queue-based background tasks and scheduled execution.

```ts
const SendEmailJob = Job.define({
  name: "send-email",
  queue: "mail",
  input: v.object({ to: v.string(), subject: v.string() }),
  async run(payload) {
    // Implementation
  },
});

const CleanupCron = Cron.define({
  name: "cleanup",
  schedule: "0 * * * *",
  async run() {
    // Scheduled logic
  },
});
```

### SDK Generation

The SDK is generated from your server definition, providing a type-safe contract for the client.

```bash
# Generated via CLI or script
aromix generate --output ./sdk.ts
```

### Client Usage

The client uses a single WebSocket connection to interact with Entities and Logic Primitives.

```ts
const client = createClient({ url: "ws://api.myapp.com" });

// Reactive Subscriptions
const users = client.entity(UserEntity);
const activeUsers = users.subscribe({ status: "active" });

activeUsers.on("change", (data) => {
  // UI updates automatically when server data changes
});

// Triggering Jobs/Workflows directly from client
await client
  .job(SendEmailJob)
  .dispatch({ to: "user@example.com", subject: "Hello" });
```

## Orchestration & Serving

The `make` function orchestrates the application, and `serve` handles the deployment using a runtime adapter.

```ts
const app = make({
  entities: [UserEntity, OrgEntity],
  workflows: [VideoWorkflow],
  jobs: [SendEmailJob],
  crons: [CleanupCron],
});

// Plugin System (Open Discussion)
// Plugins can extend the app instance via app.use() or app.decorate()
app.use(AuditLogPlugin);

// Serving via Adapter Pattern
import { serve } from "@aromix/bun";
serve(app, config);
```
